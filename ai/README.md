# Orion project knowledge

This folder is the maintained source of truth for Orion product and engineering context.

## Start here

- `project/CODEBASE.md` — where code and runtime responsibilities live.
- `project/ARCHITECTURE.md` — data flow, ownership boundaries, and failure behavior.
- `project/WORKFLOWS.md` — development, verification, build, and local installation.
- `project/WORKFLOW-DIAGRAM.md` — visual resume-work loop.
- `project/CHANGELOG.md` — significant completed structural and product changes.
- `product/PRODUCT.md` — product objective, current scope, and non-goals.
- `product/BRAND.md` — visual intent and approved brand assets.
- `features/project-focuses.md` — active outcomes, focus history, and progress behavior.
- `features/project-tasks.md` — minimal manual task behavior and deliberate non-goals.
- `features/commit-evidence.md` — expandable Git evidence and its safety boundary.
- `features/commit-impact-analysis.md` — reviewable local-AI updates from commit evidence.
- `features/repository-feature-analysis.md` — evidence-backed feature proposals from local repos.
- `integrations/lm-studio.md` — shared local AI runtime contract and configuration.

Add knowledge only when it has a durable owner: feature behavior under `features/`, external contracts under `integrations/`, engineering material under `project/`, and durable rationale under `decisions/`. Do not create empty categories for symmetry.
